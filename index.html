About
Me

Engineer. Systems thinker. Problem solver.

Healthcare Technology Data Engineering

My instinct as an engineer is to understand the system before touching the code. I've found that most performance problems aren't code problems — they're design problems. The RabbitMQ overhaul that cut ECG latency wasn't clever code, it was the right architecture applied to the right place.

I'm a Senior Backend Engineer with 6 years of experience building distributed, cloud-native systems in Java, Spring Boot, and AWS. My work spans the full backend lifecycle — from system design and API architecture to async messaging pipelines, embedded integrations, and serverless deployments.

I've worked in high-stakes domains where the software genuinely matters — where latency affects outcomes and downtime is not an option. That environment has made me precise about correctness, deliberate about trade-offs, and comfortable making architecture calls under pressure.

Outside engineering, I guide teams — I've mentored junior engineers on system design and implementation, and helped QA engineers understand infrastructure like Nginx, reverse proxies, and backend deployments. Good engineers make the people around them better.

04 Projects

Key Projects

The thinking and trade-offs behind systems I've owned end-to-end.

📡
SyncX — Patient Vitals Streaming
Healthcare · AWS Kinesis · Java · Edge Gateway
The core design challenge: how do you get reliable, low-latency data out of a moving vehicle with variable connectivity? The answer was an edge gateway that buffers locally and streams into AWS Kinesis — decoupling collection from processing so neither side blocks the other.
JavaSpring BootKinesisAWS
🆘
SOS Pendant Emergency Backend
Healthcare · Java NIO · TCP · DynamoDB · REST
Thousands of pendant devices needed persistent connections without spawning a thread per device. Java NIO's non-blocking I/O model was the right call — a single thread pool handles the selector loop while DynamoDB absorbs the write throughput for real-time location without becoming a bottleneck.
Java NIOTCPDynamoDBREST
❤️
D Assist — ECG Pipeline Overhaul
Healthcare · RabbitMQ · Java Streams · Async Architecture
REST polling for ECG data was the wrong tool for a real-time problem — the latency was a consequence of the architecture, not the code. Switching to RabbitMQ async messaging meant the frontend received data the moment it was available, not on the next poll cycle. Also built the Fluids & Medication Charts using Java Streams pipelines for time-series aggregation.
RabbitMQJava StreamsSpring BootMongoDB
🏥
Multi-Hospital Invoice Engine
Healthcare · Batch Processing · Step Functions · Fargate
Invoice generation across hospital networks is inherently a batch problem — high volume, time-insensitive, but must be auditable and fault-tolerant. Step Functions handles orchestration and retry logic, SQS decouples job dispatch from execution, and Fargate runs containerised batch jobs that scale to load without manual intervention.
Batch ProcessingLambdaSQSFargateStep Functions
📊
Client Forecasting Dashboard
Data Engineering · Node.js · Angular · PostgreSQL
The problem wasn't building a dashboard — it was replacing a manual, error-prone process with a self-serve system clients could trust. The key design decision was separating live processing (new keywords) from cached retrieval (existing ones), so the UX felt instant regardless of what was happening underneath.
Node.jsAngularPostgreSQLTypeORM

ESP32 Medical Data Bridge
Healthcare · Embedded C · ESP32 · Hardware
The existing setup required a laptop at every bedside — expensive, fragile, and hard to scale. The ESP32 replaced it entirely: collect directly from the device, transmit over the network, no laptop in the loop. Sometimes the right architectural decision is a hardware one.
